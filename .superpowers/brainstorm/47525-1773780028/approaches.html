How should the pages be structured?

Three approaches for expanding the Health Insurance and Digital Nomad pages

A

Inline page expansion

Keep the current page.tsx structure. Add country-by-country sections directly in the file with headings, general guidance paragraphs, and practical tips. No new components needed — same pattern as the existing stub pages but fully written out.

Pros

  • Simplest to implement
  • No component changes
  • Fast to ship

Cons

  • No cross-links to currency conversion
  • Less consistent with expat guides
B

Article-style (ArticleBody pattern)

Convert both pages to use the existing ArticleBody.tsx component (same pattern as the expat guides). Content defined as typed sections in TypeScript, rendered with the article island that supports inline currency conversion.

Pros

  • Consistent with guide pages
  • Gets inline USD/EUR/GBP currency toggle for free
  • Structured content type

Cons

  • More setup work
  • Currency conversion less relevant for these topics
C

Hybrid: Overview + country sections

Short general intro, then one collapsible/accordion section per country — each with 3–5 bullet points covering the key info. Uses inline page.tsx (no new components) but structured as a scannable reference rather than a long-form article.

Pros

  • Scannable for nomads comparing countries
  • Compact — users don't read top-to-bottom
  • No new components

Cons

  • Less depth per country
  • Accordion requires client component
